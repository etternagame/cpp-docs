11 // PrimaryMenuButton and SecondaryMenuButton are used to support using

14 // A button being a primary menu button means that this GameButton will generate
15 // a the corresponding MenuInput IF AND ONLY IF the GameButton corresponding to
16 // the pimary input is not mapped.
17 
18 // Example 1: A user is using an arcade machine as their controller. Most
19 // machines have MenuLeft, MenuStart, and MenuRight buttons on the cabinet, so
20 // they should be used to navigate menus. The user will map these DeviceInputs
21 // to the GameButtons "MenuLeft (optional)", "MenuStart", and "MenuRight
22 // (optional)".
23 
24 // Example 2: A user is using PlayStation dance pads to play. These controllers
25 // don't have dedicated DeviceInputs for MenuLeft and MenuRight. The user maps
26 // Up, Down, Left, and Right as normal. Since the Left and Right GameButtons
27 // have the flag FLAG_SECONDARY_MENU_*, they will function as MenuLeft and
28 // MenuRight as long as "MenuLeft (optional)" and "MenuRight (optional)" are not
29 // mapped.
